## What Can Be Contributed

SpecGantry consists primarily of:

- **SKILL.md files** — skill prompt definitions under `.claude/skills/`
- **CLAUDE.md** — the engagement contract and lifecycle governance configuration
- **SPECS/** — project artifacts and component specifications (these are project-specific and generally not contributed back)
- **Documentation** — README, NOTICE, and other supporting files

Contributions are welcome for:

- Bug fixes in skill logic or prompt structure
- Improvements to lifecycle governance rules in `CLAUDE.md`
- New skills that fit the project lifecycle model
- Documentation improvements
- Typo and clarity fixes

---

## How to Contribute

1. **Open an issue first** for anything beyond a trivial fix. Describe the problem or improvement you have in mind. This avoids duplicate effort and allows for discussion before you invest time writing.

2. **Fork the repository** and create a branch for your change.

3. **Make your changes.** Keep changes focused — one concern per pull request.

4. **Submit a pull request** with a clear description of:
   - What changed and why
   - Any trade-offs or alternatives you considered
